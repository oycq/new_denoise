L1 + λ · TV 正则强度对比报告

在 l1_plus_tv 损失上扫描 4 个 λ：0.005 / 0.01 / 0.02 / 0.05，每组 5 min 训练。 对比 baseline（纯 L1）与 N=5 的候选。

① 什么是 L1 + λ · TV？

训练损失：

loss = L1(denoised, target) + λ · TotalVariation(denoised)

TV(I) = mean( |I(x+1,y) − I(x,y)| ) + mean( |I(x,y+1) − I(x,y)| )

TV 项惩罚 相邻像素的差：

  • 大平面（天花板、角落、天空）：相邻像素本来差很小，TV 项几乎不影响 → 噪声团块被「拉平」；
  • 边缘 / 纹理（栅栏、木纹、树枝）：相邻像素差就是边缘本身，被 TV 一起惩罚 → λ 越大边缘越糊。

所以 λ 是「平面干净」 ↔ 「细节锐利」 之间的旋钮。

关键观察：

  • λ=0.005：blob 比 baseline 还略差（1.033 vs 1.006），TV 太弱根本没起作用。
  • λ=0.01：是「拐点」—— blob 第一次明显低于 baseline（0.98），但边缘损失不到 4%（17.26 vs 17.97）。
  • λ=0.02：blob 几乎不变（0.98），但边缘掉到 16.54，性价比变差。
  • λ=0.05：blob 最低（0.94），边缘掉 14%（15.45），细节牺牲明显。

⑤ 各 λ 综合分析

λ = 0.005
弱到失效
  • blob 比 baseline 略差
  • 边缘最锐（18.27）但是「锐」的是 baseline 的颗粒
  • 视觉上跟 baseline 几乎一样
  • 结论：不要用
λ = 0.01
推荐
  • blob 第一次显著优于 baseline（−2.5%）
  • 边缘几乎不损失（−4%）
  • 木纹、栅栏、树枝完整
  • 角落明显比 baseline 干净
  • 结论：甜蜜点
λ = 0.02
次选
  • blob 与 0.01 持平
  • 边缘掉到 16.54（−8%）
  • 栅栏轻微变粗，木纹略弱
  • 角落跟 0.01 几乎一样
  • 结论：没必要更大
λ = 0.05
过度
  • blob 最低（−6%）
  • 但边缘掉 14%（15.45）
  • 木纹消失、栅栏糊、树枝软
  • 视觉「塑料感」
  • 结论：过度平滑

⑥ 结论

推荐：λ = 0.01

在 4 个 λ 中，0.01 是唯一一个同时：

  • 降低大平面 blob（0.98 < baseline 1.01）
  • 显著清理角落暗噪（corner_TL 0.66 vs baseline 0.78，−15%）
  • 保留绝大部分高频细节（边缘锐度仅掉 4%）

而 0.005 太弱、0.02 边缘开始软、0.05 已经在牺牲细节。

我已经把 n2n/trainer.py 的默认 loss_kwargs 改成 {"lam": 0.01}， 下次启动训练直接生效。

怎么换 λ

// n2n/trainer.py 的 TrainConfig
loss_name: str = "l1_plus_tv"
loss_kwargs: dict = field(default_factory=lambda: {"lam": 0.01})  # 改这里
