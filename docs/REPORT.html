🏆 推荐策略

10_ms_charbonnier_p256   Multi-scale Charbonnier (4 scales) + patch 256
最低 blob 残差 + 最高边缘锐度 + 角落干净 + 无过度平滑

1. 实验设置

  • 公平条件：每个策略训练 5 分钟（300 秒），相同数据 / 优化器 / FP16，仅改 loss / 模型 / 预处理
  • 推理参考图：train_data/Data/changqiao_qiaodi/16/sensor2_raw.png（场景最高 ISO，噪声最强）
  • 输出：experiments/<name>/sensor2_isp_with_denoise.png，可在 ImageJ 作 stack 闪烁对比
  • 评估方法：
    1. 量化指标：blob_residual（低频带通能量，越低越好）+ noise_floor（高频残差，越低越好）+ edge_sharpness（Sobel 梯度均值，越高越好）
    2. 视觉评估：直接读图，对天花板/地板/四角/窗格 ROI 主观比对
    3. composite = 0.5 · (1 − blob_norm) + 0.25 · (1 − hf_noise_norm) + 0.25 · edge_norm
  • ROI 评估特别覆盖了图像四角（lens shading 重灾区）

4. 视觉评估（top 5 ROI 对比）

每张图：左边是 NOISY 原图，后面依次是 baseline / l1+TV / ms_charb_p256 / multiscale_l1 / ms4_p256 的去噪结果，全部 2× 最近邻放大。

主观判读要点

  • 13_l1_plus_tv：metric 最高，但 edge_sharpness 仅 15.45（最低），肉眼明显过度平滑——窗格、树叶细节被吃掉，看起来"水彩化"。不推荐。
  • 10_ms_charbonnier_p256：blob 第 2 低、edge sharp 18.3（中上）、hf 略高但视觉上不明显。天花板、角落、地板都干净，窗格细节保留好。✅
  • 03_multiscale_l1：与 10 接近但 patch 较小，效果稍差。
  • 09_ms4_p256、14_unet4_ms_p256：edge 很锐但 hf_noise 偏高（保留了一些颗粒）。
  • 02_l1_plus_grad：metric 倒数第 1，把高频噪声当边缘保留下来——确认最差。

5. 关键发现

  1. 多尺度 loss 是最大单一胜利点。top 6 中 4 个用了 multiscale。低频 blob 是 5 分钟训练里最难抹掉的伪影，多尺度 L1 在 1/2、1/4、1/8 下采样图上分别施加约束，从根本上压制 blob。
  2. patch 256 > patch 192（在 multiscale 类策略中 +5–10% 综合分）。更大 patch → 训练 RF / 测试 RF 更接近，泛化更好。
  3. Charbonnier 单用帮助不大，但与 multiscale 组合（10）反而是 top 候选，因 ε 平滑了 multiscale 在多分辨率上的中位数化趋势。
  4. 4 层 UNet 没明显赢面（07、14、15 都没拿到 top 3）。3 层 UNet + patch 256 已足够；更深网络在 5 min 训练步数内不足以收敛。
  5. Black level 减输入（12）几乎无变化 → FP16 不是瓶颈。
  6. TV (13) 太激进：blob/noise 指标都极低，但代价是细节全丢。可考虑 lam=0.01 而非 0.05。
  7. 角落 lens-shading 区：在 top 4 策略中都比 baseline 干净不少；multiscale 类在角落 blob 抑制最明显。

6. 最终推荐

🏆 10_ms_charbonnier_p256 — Multi-scale Charbonnier + patch 256

TrainConfig(
    loss_name="ms_l1_charbonnier",
    loss_kwargs={"scales": 4, "eps": 1e-3},
    patch_size=256,
    batch_size=6,
    base_channels=48,
    unet_depth=3,
    lr=2e-4,
    use_fp16=True,
)

这套配置已写为 n2n/trainer.py::TrainConfig 的默认值（commit c2e4176）。直接运行：

python run_train.py 1800   # 30 分钟训练 + 全量推理到 result/

为什么是它

  • blob_residual = 0.954（top 2，仅次于过度平滑的 TV）
  • edge_sharpness = 18.32（top 4，比 baseline 还好）
  • hf_noise = 1.075（与同档次相当，视觉无差）
  • 视觉上：天花板/地板/四角全部干净，窗格 / 树叶 / 木纹细节保留良好；没有 TV 的水彩化 / l1+grad 的颗粒残留

比 baseline_l1 的胜处

  • 角落 blob 显著减少（lens-shading 区干净得多）
  • 平坦区 banding 消失
  • 边缘锐度 +1.5%

备选：03_multiscale_l1

如果不想动 patch size，纯 loss 替换即可：

  • 把 loss_name="l1" 改成 loss_name="multiscale_l1"
  • 其它配置保持现状（patch 192、batch 8）
  • composite 0.620（vs baseline 0.575），blob 也低，无侵入式升级

不推荐

  • 02_l1_plus_grad（最差）
  • 13_l1_plus_tv（过度平滑）
  • 07/14/15（4 层 UNet 在 5 min 预算内训不够）
